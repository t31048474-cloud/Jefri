# Решение проблем

## Проблема: Кнопка "Зарегистрироваться" не работает

### Решение (уже исправлено в текущей версии)
Проблема была в том, что обработка ошибок при регистрации была неполной. 

**Что было сделано:**
1. Добавлена правильная обработка ответа API в `lib/auth-store.ts`
2. Теперь ошибки корректно парсятся и отображаются пользователю
3. Добавлено логирование для отладки (с префиксом `[v0]`)

### Как тестировать регистрацию
```bash
# 1. Откройте приложение
http://localhost:3000

# 2. Нажмите "Зарегистрироваться"
# 3. Заполните форму:
# - Имя: Тестовое имя
# - Email: test@example.com
# - Пароль: password123 (минимум 6 символов)

# 4. Нажмите "Создать аккаунт"
# 5. Должны войти в библиотеку
```

## Проблема: Модальное окно входа не появляется

### Решение
Убедитесь, что:
1. Cookies включены в браузере
2. localStorage не переполнен
3. Сессия действительно удалена

### Как очистить данные
```javascript
// Откройте DevTools (F12) и выполните:
localStorage.clear();
sessionStorage.clear();
document.cookie = 'session=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;';
location.reload();
```

## Проблема: Обложки книг не загружаются

### Решение
Обложки загружаются из Open Library API. Если они не загружаются:

1. Проверьте интернет-соединение
2. Убедитесь, что Open Library API доступен:
```bash
curl -I "https://covers.openlibrary.org/b/id/1234567-M.jpg"
```

3. Проверьте ISBN в файле `components/library.tsx`
4. Убедитесь, что правильные ISBN используются

### Как добавить свои обложки
Отредактируйте `components/library.tsx` и замените ISBN на нужные ISBN:
```typescript
{ 
  title: 'Название книги', 
  author: 'Автор', 
  isbn: 'НОВЫЙ_ISBN', // Замените на реальный ISBN
  rating: 4.5, 
  genre: 'Жанр' 
}
```

## Проблема: Ошибка "User already exists"

### Решение
Это нормальное поведение - вы пытаетесь зарегистрироваться с email, который уже используется.

**Варианты:**
1. Используйте другой email
2. Войдите с существующим аккаунтом
3. Удалите файл `/data/users.json` чтобы очистить всех пользователей

## Проблема: Пароль не принимается

### Решение
Пароль должен быть:
- Минимум 6 символов
- Точно такой же как при регистрации (чувствителен к регистру)

**Если забыли пароль:**
1. Удалите файл `/data/users.json`
2. Зарегистрируйтесь заново

## Проблема: Файл users.json не создается

### Решение
Файл создается автоматически при первой регистрации. Если это не происходит:

1. Проверьте права доступа к папке проекта
2. Убедитесь, что папка `/data` существует
3. Проверьте наличие ошибок в консоли браузера (F12)
4. Проверьте логи dev сервера

## Проблема: Сессия теряется после перезагрузки

### Решение
Это может быть вызвано:

1. **Cookies отключены** - включите их в настройках браузера
2. **SameSite cookies** - используется режим "lax", что должно работать
3. **Localhost и HTTPS** - для localhost HTTP достаточно, для production нужен HTTPS

### Тест сессии
```javascript
// DevTools > Console:
document.cookie // Должна показать session=...
localStorage.getItem('auth-storage') // Должен показать данные пользователя
```

## Проблема: Поиск не работает

### Решение
Поиск реализован на клиенте и работает в реальном времени. Начните вводить название книги в поле поиска в заголовке.

## Проблема: Стрелки прокрутки не видны

### Решение
Стрелки появляются при наведении мыши на секцию книг. Убедитесь, что:
1. JavaScript включен
2. CSS загружается корректно
3. Используется современный браузер

## Проблема: Приложение медленно загружается

### Решение
Первая загрузка может быть медленнее из-за:
1. Загрузки обложек из Open Library
2. Кэширования браузера
3. Скорости интернета

Последующие загрузки будут быстрее благодаря браузерному кэшу.

## Как включить отладку

Добавьте это в консоль браузера:
```javascript
// Просмотр всех логов
localStorage.setItem('debug', 'true');
location.reload();

// Просмотр cookies
document.cookie

// Просмотр localStorage
JSON.parse(localStorage.getItem('auth-storage'))

// Просмотр сетевых запросов
// Откройте DevTools > Network и перезагрузите страницу
```

## Контакт для поддержки

Если проблема не решена:
1. Проверьте логи в консоли браузера (F12 > Console)
2. Проверьте логи dev сервера (терминал)
3. Убедитесь, что все зависимости установлены: `pnpm install`
4. Попробуйте очистить кэш: `rm -rf .next node_modules && pnpm install`
