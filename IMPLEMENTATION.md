### 2. ✅ Хранение сессий в файле/БД

**Файлы:**
- `app/api/auth/register/route.ts` - регистрация пользователей
- `app/api/auth/login/route.ts` - вход пользователей
- `app/api/auth/session/route.ts` - проверка и управление сессией
- `data/users.json` - хранилище пользователей

**Механизм:**

1. **HTTP-only Cookies**
   - Используются для безопасного хранения сессии
   - Защищены от XSS атак
   - Автоматически отправляются с каждым запросом
   - Срок действия: 7 дней

2. **Bcryptjs для паролей**
   - Каждый пароль хешируется с salt=10
   - Невозможно восстановить исходный пароль
   - Безопасное сравнение паролей при входе

3. **JSON файл для хранения**
   - `data/users.json` содержит массив пользователей
   - Автоматически создается при первой регистрации
   - Содержит: id, name, email, password (хеш)

4. **Сессия в браузере**
   - Zustand хранилище для состояния на клиенте
   - localStorage для восстановления сессии после перезагрузки
   - Синхронизация с серверной сессией через cookies

### 3. ✅ Реальные обложки из Open Library

**Файлы:**
- `lib/book-service.ts` - сервис для работы с Open Library API
- `components/book-card.tsx` - отображение обложек книг
- `components/library.tsx` - основные данные (ISBN) для каждой книги

**Механизм:**

1. **ISBN как ключ**
   - Каждая книга в `BOOKS_DATA` содержит ISBN
   - ISBN уникально идентифицирует изданиюю книги

2. **URL для обложек**
   ```
   https://covers.openlibrary.org/b/isbn/{ISBN}-M.jpg
   ```
   - `M` = средний размер (300px)
   - Кешируется браузером автоматически
   - Полностью бесплатно и без ограничений

3. **Fallback механизм**
   - Если обложка не загружается, показывается иконка книги
   - Image компонент Next.js обрабатывает ошибки загрузки

4. **Информация о книге**
   - Функция `getBookFromISBN()` получает полные данные
   - Функция `getBooksFromISBNList()` для пакетной загрузки
   - Axios для HTTP запросов

## 🔌 API Endpoints

### POST `/api/auth/register`

**Request:**
```json
{
  "name": "Иван",
  "email": "ivan@example.com",
  "password": "SecurePass123"
}
```

**Response (201):**
```json
{
  "id": "abc123xyz",
  "name": "Иван",
  "email": "ivan@example.com"
}
```

**Ошибки:**
- 400: Отсутствуют поля
- 400: Пароль меньше 6 символов
- 400: Пользователь уже существует
- 500: Внутренняя ошибка

### POST `/api/auth/login`

**Request:**
```json
{
  "email": "ivan@example.com",
  "password": "SecurePass123"
}
```

**Response (200):**
```json
{
  "id": "abc123xyz",
  "name": "Иван",
  "email": "ivan@example.com"
}
```

**Ошибки:**
- 400: Отсутствуют email или пароль
- 401: Неверные учетные данные
- 500: Внутренняя ошибка

### GET `/api/auth/session`

**Response (200):**
```json
{
  "id": "abc123xyz",
  "name": "Иван",
  "email": "ivan@example.com"
}
```

**Ошибки:**
- 401: Нет активной сессии
- 401: Неверная сессия

### POST `/api/auth/session` (Logout)

**Response (200):**
```json
{
  "success": true
}
```

## 🔐 Безопасность

### Реализованные меры:

1. **HTTP-only Cookies**
   - Флаг `httpOnly: true` - JavaScript не может прочитать cookie
   - Флаг `secure: true` в production - только HTTPS
   - Флаг `sameSite: 'lax'` - защита от CSRF

2. **Bcryptjs**
   - Salt = 10 (нужно в среднем 2^10 = 1024 хеш-операции)
   - Защита от перебора паролей (brute force)
   - Невозможно восстановить пароль из хеша

3. **Валидация на сервере**
   - Все входные данные проверяются
   - Email формат не проверяется (можно добавить regex)
   - Пароль минимум 6 символов

4. **Защита от дублирования**
   - Проверка email при регистрации
   - Email должен быть уникальным

## 🔄 Поток данных

### Регистрация:
```
Form Input → AuthModal component
    ↓
useAuthStore.register()
    ↓
POST /api/auth/register
    ↓
Валидация → Хеширование → Сохранение в users.json
    ↓
Set-Cookie: session
    ↓
Zustand store update
    ↓
Modal close → Show Library
```

### Вход:
```

    ↓
useAuthStore.login()
    ↓
POST /api/auth/login
    ↓
Поиск пользователя → Проверка пароля
    ↓
Set-Cookie: session
    ↓
Zustand store update
    ↓
Modal close → Show Library
```

```
Page load
    ↓
checkSession()
    ↓
GET /api/auth/session
    ↓
Cookie exists? ──YES──→ Zustand update → Show Library
    ↓NO
Show AuthModal
    ↓
User logs in/registers
    ↓

    ↓
Show Library
```
