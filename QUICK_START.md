# ⚡ Быстрый старт

## Установка и запуск

```bash
# 1. Установить зависимости
pnpm install

# 2. Запустить dev сервер
pnpm dev

# 3. Открыть в браузере
# http://localhost:3000
```

## 🔐 Как зарегистрироваться и войти

### Первый запуск
При первом входе приложение автоматически откроет модальное окно с формой входа:

1. **Нажмите на "Зарегистрироваться"** (синяя ссылка внизу)
2. **Заполните поля:**
   - Имя: ваше имя
   - Email: your@email.com
   - Пароль: минимум 6 символов
3. **Нажмите кнопку "Создать аккаунт"**
4. Вы будете авторизованы и перенесены в библиотеку

### Вход в существующий аккаунт
1. **Заполните поля:**
   - Email: ваш email
   - Пароль: ваш пароль
2. **Нажмите "Войти"**

### Выход из аккаунта
Нажмите на кнопку выхода (иконка с дверью) в правом верхнем углу рядом с вашим именем.

## 🔄 Как работает система

```
┌──────────────────────────────────────────────────────┐
│ 1. Загрузка приложения                               │
│    ↓                                                  │
│ 2. Проверка сессии (checkSession)                    │
│    ↓                                                  │
│ Сессия найдена? ──ДА──→ Показать библиотеку         │
│    ↓НЕТ                                              │
│ 3. Показать модальное окно входа                    │
│    ↓                                                  │
│ 4. Пользователь заполняет форму                      │
│    ↓                                                  │
│ 5. Отправить данные на /api/auth/register или login │
│    ↓                                                  │
│ 6. Сервер:                                           │
│    - Валидирует данные                               │
│    - Хеширует пароль (bcryptjs)                      │
│    - Сохраняет пользователя в data/users.json       │
│    - Устанавливает HTTP-only cookie                  │
│    ↓                                                  │
│ 7. Клиент сохраняет пользователя в Zustand         │
│    ↓                                                  │
│ 8. Закрыть модальное окно и показать библиотеку     │
└──────────────────────────────────────────────────────┘
```

## 📖 Работа с книгами

### Где хранятся книги?
Файл: `components/library.tsx` → объект `BOOKS_DATA`

### Структура одной книги:
```typescript
{
  title: 'Война и мир',           // Название
  author: 'Лев Толстой',          // Автор
  isbn: '0199232768',             // ISBN для обложки
  rating: 4.8,                    // Рейтинг (0-5)
  genre: 'Романы'                 // Жанр
}
```

### Как найти правильный ISBN?
1. Перейдите на [Open Library](https://openlibrary.org/)
2. Найдите книгу в поиске
3. На странице найдите ISBN и скопируйте его
4. Замените в коде

### Обложка загружается по URL:
```
https://covers.openlibrary.org/b/isbn/{ISBN}-M.jpg
```

## 🔐 Безопасность

✅ **Сделано:**
- HTTP-only cookies (защита от XSS)
- Хеширование паролей bcryptjs
- Валидация на сервере
- Проверка пароля при входе

⚠️ **Не для production:**
- Пользователи хранятся в JSON файле
- Для production используйте Neon или Supabase

## 🐛 Решение проблем

### Модальное окно не закрывается
- Проверьте консоль браузера (F12 → Console)
- Убедитесь, что все поля заполнены корректно
- Пароль должен быть минимум 6 символов

### Обложки не загружаются
- Проверьте ISBN (должен быть правильным)
- Откройте в браузере: `https://covers.openlibrary.org/b/isbn/{ISBN}-M.jpg`
- Если обложка не найдена, выберите другой ISBN

### Сессия теряется при обновлении страницы
- Проверьте, что cookies включены
- Убедитесь, что http://localhost:3000 доступен
- Очистите localStorage: `localStorage.clear()`

## 🚀 Развертывание

Приложение готово для развертывания на Vercel:

```bash
# 1. Инициализировать git репозиторий
git init

# 2. Коммитить код
git add .
git commit -m "Initial commit"

# 3. Развернуть на Vercel
# Используйте v0 Deploy или Vercel CLI
```

⚠️ **Важно для production:**
- Замените `data/users.json` на реальную БД (Neon/Supabase)
- Добавьте HTTPS
- Используйте environment variables для секретов
- Установите BETTER_AUTH_SECRET для сессий
